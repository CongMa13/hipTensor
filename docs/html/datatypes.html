hipTENSOR Data Types

hiptensorStatus_t

enum hiptensorStatus_t

hipTENSOR status type returns

The type is used for function status returns. All hipTENSOR library functions return their status, which can have the following values.

Values:

enumerator HIPTENSOR_STATUS_SUCCESS

The operation completed successfully.

enumerator HIPTENSOR_STATUS_NOT_INITIALIZED

The opaque data structure was not initialized.

enumerator HIPTENSOR_STATUS_ALLOC_FAILED

Resource allocation failed inside the hipTENSOR library.

enumerator HIPTENSOR_STATUS_INVALID_VALUE

An unsupported value or parameter was passed to the function (indicates an user error).

enumerator HIPTENSOR_STATUS_ARCH_MISMATCH

Indicates that the device is either not ready, or the target architecture is not supported.

enumerator HIPTENSOR_STATUS_MAPPING_ERROR

An access to GPU memory space failed, which is usually caused by a failure to bind a texture.

enumerator HIPTENSOR_STATUS_EXECUTION_FAILED

The GPU program failed to execute. This is often caused by a launch failure of the kernel on the GPU, which can be caused by multiple reasons.

enumerator HIPTENSOR_STATUS_INTERNAL_ERROR

An internal hipTENSOR error has occurred.

enumerator HIPTENSOR_STATUS_NOT_SUPPORTED

The requested operation is not supported.

enumerator HIPTENSOR_STATUS_LICENSE_ERROR

The functionality requested requires some license and an error was detected when trying to check the current licensing.

enumerator HIPTENSOR_STATUS_CK_ERROR

A call to CUBLAS did not succeed.

enumerator HIPTENSOR_STATUS_ROCM_ERROR

Some unknown HIPTENSOR error has occurred.

enumerator HIPTENSOR_STATUS_INSUFFICIENT_WORKSPACE

The provided workspace was insufficient.

enumerator HIPTENSOR_STATUS_INSUFFICIENT_DRIVER

Indicates that the driver version is insufficient.

enumerator HIPTENSOR_STATUS_IO_ERROR

Indicates an error related to file I/O.

hiptensorDataType_t

enum hiptensorDataType_t

hiptensorDataType_t is an enumeration of the types supported by HIPTENSOR libraries. hipTENSOR supports real FP16, BF16, FP32 input types. FP64 is not supported by the xdlops compiler issue (SWDEV-335738)

Values:

enumerator HIPTENSOR_R_16F

real as a half

enumerator HIPTENSOR_R_16BF

real as a nv_bfloat16

enumerator HIPTENSOR_R_32F

real as a float

enumerator HIPTENSOR_R_64F

real as a double

hiptensorComputeType_t

enum hiptensorComputeType_t

Encodes hipTENSOR’s compute type.

Values:

enumerator HIPTENSOR_COMPUTE_16F

floating-point: 5-bit exponent and 10-bit mantissa (aka half)

enumerator HIPTENSOR_COMPUTE_16BF

floating-point: 8-bit exponent and 7-bit mantissa (aka bfloat)

enumerator HIPTENSOR_COMPUTE_TF32

floating-point: 8-bit exponent and 10-bit mantissa (aka tensor-float-32)

enumerator HIPTENSOR_COMPUTE_32F

floating-point: 8-bit exponent and 23-bit mantissa (aka float)

enumerator HIPTENSOR_COMPUTE_64F

floating-point: 11-bit exponent and 52-bit mantissa (aka double)

enumerator HIPTENSOR_COMPUTE_8U

8-bit unsigned integer

enumerator HIPTENSOR_COMPUTE_8I

8-bit signed integer

enumerator HIPTENSOR_COMPUTE_32U

32-bit unsigned integer

enumerator HIPTENSOR_COMPUTE_32I

hiptensorAlgo_t

enum hiptensorAlgo_t

Allows users to specify the algorithm to be used for performing the tensor contraction.

This enum gives users finer control over which algorithm should be executed by hiptensorContraction(); values >= 0 correspond to certain sub-algorithms of GETT.

Values:

enumerator HIPTENSOR_ALGO_DEFAULT_PATIENT

Uses the more accurate but also more time-consuming performance model.

enumerator HIPTENSOR_ALGO_GETT

Choose the GETT algorithm.

enumerator HIPTENSOR_ALGO_TGETT

Transpose (A or B) + GETT.

enumerator HIPTENSOR_ALGO_TTGT

Transpose-Transpose-GEMM-Transpose (requires additional memory)

enumerator HIPTENSOR_ALGO_DEFAULT

Lets the internal heuristic choose.

hiptensorWorksizePreference_t

enum hiptensorWorksizePreference_t

This enum gives users finer control over the suggested workspace.

This enum gives users finer control over the amount of workspace that is suggested by hiptensorContractionGetWorkspace

Values:

enumerator HIPTENSOR_WORKSPACE_MIN

At least one algorithm will be available.

enumerator HIPTENSOR_WORKSPACE_RECOMMENDED

The most suitable algorithm will be available.

enumerator HIPTENSOR_WORKSPACE_MAX

All algorithms will be available.

hiptesnorContractionOperation_t

enum hiptesnorContractionOperation_t

This enum decides the operation based on the in puts passed in the hiptensorContractionGetWorkspace

Values:

enumerator HIPTENSOR_CONTRACTION_SCALE

\({C=\alpha\mathcal{A}\mathcal{B}}\)

enumerator HIPTENSOR_CONTRACTION_BILINEAR

\({D=\alpha\mathcal{A}\mathcal{B}+\beta\mathcal{C}}\)
